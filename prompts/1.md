# 개요
시나리오 작가를 위한 플롯, 시나리오 텍스트 에디터를 개발하기 위함

## 기능
1. 인물간의 관계 시각화
2. **플롯**이라는 단위로 나누어 시나리오를 관리
  - 예를 들어 하나의 워드 파일이 아닌 플롯 1, 플롯 2, ..., 플롯 n으로 각 플롯을 관리함
  - 각 플롯 카드를 클릭하면 해당 플롯을 수정 가능
  - 플롯은 제목, 내용을 가짐
  - mouse로 drag & drop하여 플롯의 순서 수정 가능 (당연히 플롯 n에서의 n 숫자도 그에 맞춰 변경되어야 함)
3. 희곡 대본 작성시 편의 기능
  - 원래 Microsoft Word 같은 에디터에서 대사를 작성하려면 "안나: 안녕하세요. (줄넘김) 브론스키: 안녕하십니까" 를 직접 타이핑해야 함.
  - 인물 이름, 콜론(:)과 같이 반복적으로 사용되는 부분들을 일일이 타이핑하고 싶지 않음
  - 또한 텍스트 서식도 자동으로 맞춰주어야 함
    - 인물과 대사 사이에는 일정 간격의 여백이 항상 유지된다.
  - [인물: 대사]라는 한 세트가 색깔로 구분지어졌으면 함.
    - 안나의 이름과 대사는 연보라색, 브론스키의 이름과 대사는 초록색 등
  - 지문, 나레이션 등 희곡에서 사용하는 특수 clause도 각자 서식을 가지면 좋겠음
  - 사용자가 Enter 입력할 때마다 직전 line에서 작성하던 서식을 복사해서 사용하고 Enter 한 번 더 입력시 새로운 서식 선택하도록 하고 싶음.
  - 이는 Notion, Obsidian 등 markdown editor를 생각하면 쉬울 것이다.


## 결정해야 하는 사항
1. 로컬 텍스트 에디터 application으로 만들어서 공유하고 싶은데 가능할지
  - 가능하다면 어떤 기술 스택 사용해야 하는지
  - 불가능하다면 대안으로 web app으로 개발 예정
