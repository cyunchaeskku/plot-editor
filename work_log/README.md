# Work Log 작성 규칙

날짜별 `.md` 파일로 작업 내용을 기록합니다.

## 파일 명명 규칙
`YYYY-MM-DD.md` — 같은 날 두 번째 세션부터는 `YYYY-MM-DD_2.md`, `YYYY-MM-DD_3.md` 형식으로 suffix를 붙인다.
날짜는 실제 날짜 기준이며, 새 세션이라도 날짜가 바뀌지 않으면 날짜를 올리지 않는다.

## 로그 형식

```markdown
# YYYY-MM-DD

## 작업 내용
- 완료한 것들

## 결정 사항
- 구현 중 결정한 것들

## 다음 작업
- 이어서 할 것들

## 이슈/메모
- 트러블슈팅, 참고사항
```
