## 작업 내용

### 1. work_log 파일 명명 규칙 정리

- `README.md` 규칙 수정: 같은 날 여러 세션은 `YYYY-MM-DD_2.md`, `_3.md` suffix 사용. 날짜는 실제 날짜 기준으로 새 세션이어도 날짜를 올리지 않음.
- 잘못 생성된 파일 이름 수정:
  - `2026-03-02.md` → `2026-03-01_2.md`
  - `2026-03-03.md` → `2026-03-01_3.md`
  - `2026-03-04.md` → `2026-03-01_4.md`

### 2. 대댓글 YouTube 스타일 thread line 적용 (`ThreadDetail.tsx`)

- 기존: 대댓글에 `ml-6 mt-2` 단순 들여쓰기
- 변경: 부모 댓글 아바타(22px) 중심선(11px) 기준 `border-l-2` 세로 선으로 연결
  - replies + reply input을 하나의 thread 컨테이너로 통합
  - `ml-[11px] border-l-2 border-gray-400 pl-3 flex flex-col gap-3`
  - depth=1 아이템의 자체 margin 제거, 부모 컨테이너가 indentation 담당
  - reply input도 thread 선 안에 포함되어 일관된 구조

### 3. 404 on refresh 원인 분석

- 원인: `createBrowserRouter` (HTML5 History API) 사용으로 `/thread` 새로고침 시 서버가 해당 경로 파일을 찾지 못함
- 환경별 영향: Tauri 빌드, S3/CloudFront 정적 호스팅에서 발생. Vite dev server는 기본 SPA fallback 처리.
- 수정 미적용 — 실행 환경 확인 필요
